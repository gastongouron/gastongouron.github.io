Ruby: Object collections

Arrays & Hashes

About collections in Ruby

In ruby, and probably wider in programming, there are many reasons why you will want to find, filter, organize and perform plenty other operations on objects that match a certain criteria. Ruby allows us to create those collections by letting us use “containers”. In ruby, the most common containers in which we will store that data and make it an accessible are the two built-in classes hashes and arrays.

An array is like a freight train!

An array is an ordered collection of objects. That may sound weird, but my understanding or array can be compared to trains. The name of the array is the engine, and the objects contained inside it the wagons.

 engine = [wagon_1, wagon_2, wagon_3]

In this collection structure you will be able to access every object of the collection using its index. The index is an integer that represents the location of the object in the array (remember we start counting at 0, so the first item of your array will be array=[0] ). An array can contain a multiple kind of objects such as integers, floats, strings, math operation, classes and even arrays!

 wagon_1 = 'cat'
 wagon_2 = 13.37
 wagon_3 = (7*3).to_i

 engine = [wagon_1, wagon_2, wagon_3]
 engine.push([])  #=> ['cat', 13.37, 21, []]

The interest of using an array is that you can access a large library of built-in methods in order to process the data of the array the way you want, adding (or concat), extracting (shift and pop), sorting, querying and transforming them in many different ways.

 engine[2]  #=> 21

For more infos about arrays built-in methods, see Official Arrays Ruby docs.

An hash is like an array...

Like arrays, hashes are collections of objects but with some extra purposes. In fact, they works with a key/value pair principle that allows you to perform more complex filtering and sorting operations than with arrays. In hashes, the left value is connected to the right value with the => operator. Let's see what our engine array would look like if it was an hash:

 engine_hash = {
   'wagon_1' => 'cat',
   'wagon_2' => 13.37,
   'wagon_3' => (7*3)
 }

Again, like arrays, hashes can be created, ordered and used in many different ways, they will become very handy every time you will have to work with informations that come in a key => value model because you can retrive the value simply by using the key, without iterating!

 engine_hash['wagon_1'] #=> 'cat'

For more infos about hashes built-in methods, see Official Hashes Ruby docs.
