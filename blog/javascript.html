JS/Ruby: A tale of loops

Is looping in ruby the same as in JS?

What is a loop?

In Ruby and JavaScript, as in computer programming in general, a loop is a sequence of instruction s that is continually repeated until a certain condition is reached. Typically, a certain process is done, such as getting an item of data and changing it, and then some condition is checked such as whether a counter has reached a prescribed number. Basically, a loop is a fundamental programming idea that is commonly used in writing programs and, Ruby and JS offers similar ways to perform those loop but with a slightly different syntax because of their internal logic.

In the next part of the article we will try to understand basic ruby loops and explain how they work internally, then, those loops will be "translated" into their jasvascript equivalent notation in order to compare both languages ways of looping.

Some Ruby loops: while, until and for

One of the main idea with the most common Ruby loops (while, until...) is that they can be used for both a statement, and a modifier. For example, if we consider the while ruby method (which executes code while conditional is true), we see it can be used for both purposes: 

 # As a statement
 while condition  [do]
   #code
 end

 # As a modifier
 code while condition
 # Or (as a modifier too)
 begin
   code
 end while conditional

The second example executes our code while conditional is true and we can have the same usage of the until (Executes code while conditional is false.) such as:

 # As a statement
 until conditional [do]
   code
 end

 # As a modifier
 code until conditional
 # Or
 begin
   code
 end until conditional

So basically the only difference between our two first kind of loops is that one processes code block when the condition evaluates to true and the other to false. I belive this is the reason why Ruby came up with a third kind of frequently used loop, the for (Executes code once for each element in expression.). Contrary to while and until, for can only be used as a statement. Nevertheless, it exists a keyword break you can use in order to stop the loop:

 # Simple for loop
 for variable [, variable ...] in expression [do]
   code
 end

 # Loop stopped with a break
 for i in 0..5
   if i > 2 then
      break
   end
   puts "Value of local variable is #{i}"
 end

Even if there are some other looping methods used in ruby such as next, redo and retry I think those three might be the one you will see the more often. Now you have a little overview of some different kinds of Ruby loops and it's time to compare these to JS's ones.

Loops in JavaScript: while, for and do/while

Even if the most common way to create conditional executions (multiple at least) is if (just like in Ruby), while, for and do...while are common ways to create a conditionnal statement execution in JavaScript or to repeat some actions. We call those loops, because, unless a condition if filled, they will force your script to keep on repeating a certain operation. For and while are very good ways to increment a value until a certain point because In JavaScript they can be used to internally define a variable and increment/decrement it:

 // for:
 var count = 10;
   for (var i = 0; i < count; i++) {
     document.writeln("looping away!");
   }

 // While:
 var count = 0;
 while (count < 10) {
   document.writeln("looping away!");
   count++;
   }

As you can see both while and for are doing the same job. The only difference is that for the for loops I used an extra argument i++ in order to increment the i value contrary to the while loop where the count++ used as an argument is a part of the loop and not of the arguments of the function while.

Finally, the do...while loop can be used for the same purposes and is structurally working as both a statement and a modifier: when do is asking the program to perform a task (incrementing a count for example), the program executes then will find a modifier while in order to repeat the condition until a certain conditionnal.

 var count = 0;
 do {
     count++;
     document.writeln("looping away!");
 } while (count < 10);

Even if Ruby and JavaScript doesn't not rely on the same type of objects and the same notions of classes the concept of looping remains the same between the two languages: incrementing a value/wait until a condition is reached in order to perform an action a certain amount of times (or until this condition is reached). This is the reason why both languages offer tailored structures of loops, and why you have to choose between them wisely referring to your needs!
