Ruby: Iterate through an enumerable with .cycle

Enumerable#cycle

Cycle, a method of Enumerable in Ruby

In ruby, Enumerable provides many useful methods to help us when working on a collection. Cycle is one of them. In the simplest possible terms, for given a collection, cycle creates an infinitely-repeating ordered source of its collection items. This method expects an optional code block to act on the items in the infinitely repeating until a break statement is encountered. If no block is given, cycle returns an Enumerator that acts as the infinite source of the collection items..

Cycle and arrays

If you use cycle on an array, and provide a code block, cycle will pass every single item into this block. When the last item has been passed, cycle will simply start again at the beginning of the array and this, until the end of time (or until cycle encounters a break statement).

To illustrate the power of cycle, let’s create a days_of_the_week array that holds every day of the week as a string and see what happens when we invoke cycle on it:

 days_of_the_week = ["Monday", "Tuesday", "Wednesday",
 "Thursday", "Friday", "Saturday", "Sunday"]

 days_of_the_week.cycle {|day| puts day}
 => Monday
 Tuesday
 Wednesday
 Thursday
 Friday
 Saturday
 Sunday
 Monday
 Tuesday
 …
 # This will print the days of the week forever.
 # Or until you stop it with CTRL+C.

In fact, you will hardly ever need an infinite repetition of your days of the week, so you may need a “break” statement in order to break the cycle. Let's see how it works: 

 days_of_week.cycle do |day|
    puts day
    break if day == “Wednesday”
 end

 => Monday
 Tuesday
 Wednesday
 => nil

The break statement is stopping .cycle when the condition == "Wednesday" is met. Also, we can notice .cycle returns nil if the loop has finished without getting interrupted.

Cycle and hashes

When used on a hash and a block is provided, .cycle will collect, map and pass each key/value pair in the hash to the block. So, each key/value pair is passed to the block, where the operation in the block is performed on the item. Once the last hash item has been passed to the block, .cycle starts over again at the beginning of the hash. .cycle goes through the hash forever (unless it encounters a break statement in the block).

To illustrate the power of .cycle on an hash, let’s make a normal_day_hash that represents what I do everyday for few days:

 normal_day_hash = {
 :morning => "wake up early and read the WGR",
 :noon => "take a lunch break",
 :afternoon => "work on DBC challenges.",
 :evening => "try to cook a good diner",
 :night => "write some ruby before I go to bed" }

By passing the argument 2 to the method cycle, the method is going to cycle 2 times through the hash. For every cycle, every key/value pair of the hash will be treated by the block and return my daily routine of the next two days as a string. So, here are my plans for the next two days:

 normal_day_hash.cycle(2) {|time, activity|
 puts "On the #{time}: I #{activity}"}

 => On the morning: I wake up early and read the WGR
 On the noon: I take a lunch
 On the afternoon: I work on DBC challenges.
 On the evening: I try to cook a good diner
 On the night: I write some ruby before I go to bed
 On the morning: I wake up early and read the WGR
 On the noon: I take a lunch break
 On the afternoon: I work on DBC challenges.
 On the evening: I try to cook a good diner
 On the night: I write some ruby before I go to bed
 => nil
